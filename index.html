An Educational Study Sheet for Programming Foundations

üéØ Introduction

Scope determines where a variable can be accessed in your code, while Lifetime determines how long a variable exists in memory. Understanding these concepts is crucial for writing efficient and bug-free C++ programs!

üîë Key Concepts

üåçGlobal Scope

Variables declared outside all functions. Accessible throughout the entire program.

üè†Local Scope

Variables declared inside functions or blocks. Only accessible within that context.

‚è∞Automatic Lifetime

Variables created when block starts, destroyed when block ends.

üîíStatic Lifetime

Variables that persist throughout program execution.

üíª Global vs Local Variables

// Global Variable int globalVar = 100; void displayGlobal() { cout << "Global: " << globalVar; // ‚úÖ Accessible } int main() { // Local Variable int localVar = 50; cout << globalVar; // ‚úÖ Global accessible cout << localVar; // ‚úÖ Local accessible displayGlobal(); // ‚úÖ Works // cout << localVar; // ‚ùå Error in displayGlobal() return 0; }
üí° Best Practice: Minimize use of global variables. They can lead to unexpected behavior and make debugging difficult!

üìä Scope Visualization

Variable Accessibility Levels

üåç Global Scope - Accessible everywhere
üè† Function Scope - Accessible in function only
üì¶ Block Scope - Accessible in block { } only

üì¶ Block Scope in Action

int main() { int x = 10; // Function scope if (x > 5) { int y = 20; // Block scope cout << x; // ‚úÖ x accessible cout << y; // ‚úÖ y accessible } cout << x; // ‚úÖ x still accessible // cout << y; // ‚ùå Error: y out of scope return 0; }

üîí Static Variables - Persistent Lifetime

void counter() { static int count = 0; // Initialized once count++; cout << "Count: " << count << endl; } int main() { counter(); // Output: Count: 1 counter(); // Output: Count: 2 counter(); // Output: Count: 3 return 0; }
üéØ Key Point: Static variables retain their value between function calls. They're initialized only once!
